Example implementation — Crossing-specific. This shortcode is specific to The Crossing's background check provider history and is provided as a reference only. It includes logic for transitioning from Safe Ministry Solutions to a different provider, and references attribute keys specific to our environment. You will need to replace this with your own implementation based on your background check provider and Rock attribute configuration.
{[ bkgdcheckstatus personid:'' ]}

Parameters

Person Id
REQUIRED: The person for which to get the background check status

Output

The output is a JSON similar to the following:

    {
        "IconClass" : ["ti ti-shield-check", "ti ti-shield-check-filled"],
        "IconColor" : [based on status],
        "IconOpacity" : []"1.0", "0.4"],
        "Status" : ["None", "Pending", "Expired", "Pass", "Pass-Expired", "Fail"],
        "StatusDescription" : [description for tooltip]
    }
